# L1.5 Answer-First Q&A Implementation Plan

> **For Hermes:** Use subagent-driven-development skill to implement this plan task-by-task.

**Goal:** Make L1.5 easier to demo by showing adaptive answers at the top immediately after a question, rendering graph responses as an actual graph, and adding a briefing-free Q&A-only page.

**Architecture:** Extract the adaptive memory visualization from `VoiceBriefing` into a reusable component. Use it both inside the existing briefing route and a new `/qna/[meetingId]` route. Keep `/api/memory/query` unchanged unless a display-mode bug appears; focus on presentation and discoverability.

**Tech Stack:** Next.js App Router, React client components, TypeScript, Tailwind CSS, existing `/api/memory/query` route, existing graph payload from `MemoryQueryVisualResponse`.

---

## Requirements From Chat

Hong asked for:

1. Answer output should appear at the top so users do not need to scroll down.
2. Graph queries should actually show a graph, not only edge text/bullets.
3. Add a version/toggle without the briefing — just Q&A + adaptive table/graph/cards later.

---

## Proposed UX

### Existing briefing page

- Keep voice briefing and typed fallback.
- When `visualResponse` exists, render a full-width **Latest answer** panel above the voice orb / controls.
- Also keep the compact side panel as context, but the top answer is the primary demo surface.

### New Q&A-only page

Add:

```txt
/qna/meeting-2026-06-20-tan
```

The page contains:

- Top hero: “Ask client memory”
- Input fixed near the top
- Suggested question chips
- Latest answer panel directly under the input
- Adaptive visual area for graph/table/cards/recommendation/missing info
- No voice orb and no long opening briefing text

### Graph display

Create a reusable compact SVG graph renderer for `response.graph`:

- Nodes displayed as cards/circles at deterministic positions.
- Edges shown as SVG lines with relationship labels.
- Include small legend/summary below for mobile/readability.
